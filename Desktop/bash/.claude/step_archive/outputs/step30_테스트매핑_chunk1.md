## 3. 테스트 작성 지침

### 테스트 프레임워크
- **프레임워크:** Vitest
- **Assertion:** Vitest expect() 또는 Chai
- **Mock:** Vitest.mock()
- **커버리지:** c8 (v8 provider)

### 파일 구조
```
tests/
  ├── classes/
  │   ├── UIController.test.js
  │   ├── AnimationManager.test.js
  │   ├── ViewRenderer.test.js
  │   └── HashTable.test.js
  ├── js/
  │   ├── App.test.js
  │   ├── TutorialManager.test.js
  │   ├── Visualizer.test.js
  │   └── AnimationEngine.test.js
  ├── utils/
  │   └── constants.test.js
  └── setup.js (공통 설정)
```

### 테스트 작성 규칙

1. **파일명 규칙:** `[파일명].test.js`
2. **모듈 포맷:** ES6 (import/export)
3. **커버리지 목표:** 각 파일 80% 이상
4. **테스트 케이스 명명:** describe/it 활용

```javascript
// 예시
describe('UIController', () => {
  describe('search()', () => {
    it('should return items matching key', () => {
      // Arrange
      const controller = new UIController()

      // Act
      const result = controller.search('key')

      // Assert
      expect(result).toBeDefined()
    })
  })
})
```

### Mock 전략

| 파일 | Mock 대상 | 방법 |
|------|---------|------|
| UIController | ViewRenderer, AnimationManager | Vitest.mock() |
| AnimationManager | GSAP | 라이브러리 Mock |
| ViewRenderer | DOM, console | DOM 환경 설정 |
| HashTable | - | Mock 불필요 (순수 로직) |

### 커버리지 검사

```bash
# 작성 중 커버리지 확인
npx c8 --per-file vitest run

# 최종 커버리지 리포트
npx c8 --reporter=json --reporter=html vitest run
```
