# Step 13 조사결과: USFCA 시각화 도구 UI/UX 분석 (Chunk 4)

## 개요
USFCA(University of San Francisco)의 Data Structures Visualization 도구들을 Playwright로 조사함.
- Open Hashing (체이닝 기반 해시 테이블)
- Closed Hashing (개방 주소 지정 해시 테이블)

---

## 1. 상단 제어 영역 (Top Control Panel)

### 1.1 입력 필드
- 텍스트 입력 필드: 사용자가 정수 또는 문자열 값을 직접 입력
- 위치: 페이지 최상단, 버튼들의 왼쪽

### 1.2 주요 버튼 (Action Buttons)
```
[입력필드] [Insert] [Delete필드] [Delete] [Find필드] [Find]
```

**버튼 기능:**
- Insert: 입력 필드의 값을 해시 테이블에 삽입
- Delete: 입력 필드의 값을 해시 테이블에서 제거
- Find: 입력 필드의 값을 해시 테이블에서 검색

### 1.3 라디오 버튼 옵션 (Input Type Selection)

#### Open Hashing
```
◉ Hash Integer
○ Hash Strings
```
- 정수 또는 문자열 입력 방식 전환
- 선택된 모드에 따라 시각화 동작 변경

#### Closed Hashing
```
◉ Hash Integer          ◉ Linear Probing: f(i) = i
○ Hash Strings          ○ Quadratic Probing: f(i) = i * i
                        ○ Double Hashing: f(i) = i * hash2(elem)
```
- 왼쪽: 입력 데이터 타입 (정수/문자열)
- 오른쪽: 충돌 해결 방법 선택
  - Linear Probing: 순차적 탐사
  - Quadratic Probing: 제곱 간격 탐사
  - Double Hashing: 이중 해싱

---

## 2. 시각화 영역 (Canvas/Visualization Area)

### 2.1 Open Hashing 시각화
- **구조**: 해시 테이블 + 체이닝 (연결 리스트)
- **표현**: 버킷들이 네모 박스로 표시, 각 버킷은 대각선 표시로 채워짐
- **버킷 인덱스**: 0-12번 인덱스 표시 (기본 크기 13)
- **애니메이션**: 삽입/삭제/검색 시 버킷과 체인 연결을 시각화

### 2.2 Closed Hashing 시각화
- **구조**: 해시 테이블 (다중 행으로 표시)
- **표현**: 3행 x 11열 구조 (총 33개 슬롯)
  - 행 1: 인덱스 0-10
  - 행 2: 인덱스 11-21
  - 행 3: 인덱스 22-28
- **셀 상태**: 비어있음(공백), 값 저장(숫자 표시), 삭제됨(표시)
- **동작**: 충돌 해결 방법에 따라 다른 경로로 탐사 시각화

---

## 3. 애니메이션 제어 영역 (Bottom Control Panel)

### 3.1 애니메이션 플레이백 제어
```
[Skip Back] [Step Back] [Pause] [Step Forward] [Skip Forward]
```

**버튼 기능:**
- Skip Back: 처음으로 돌아가기
- Step Back: 한 단계 이전으로
- Pause: 애니메이션 일시 정지
- Step Forward: 한 단계 앞으로
- Skip Forward: 마지막으로 이동

**상태 표시:**
- "Animation Completed" 텍스트: 애니메이션이 완료되었음을 표시
- Pause 버튼이 중앙에 위치하여 대칭 구성

### 3.2 애니메이션 속도 설정
```
[w: 1000] [h: 500] [Change Canvas Size] [Move Controls]
```

**설정 항목:**
- w (Width): 캔버스 너비 설정 (기본값: 1000px)
- h (Height): 캔버스 높이 설정 (기본값: 500px)
- Change Canvas Size: 입력한 크기로 캔버스 재설정
- Move Controls: 제어판 위치 이동 (상단/하단 전환)

### 3.3 애니메이션 속도
- 슬라이더: 애니메이션 실행 속도 조절 (빠름/느림)
- 레이블: "Animation Speed" 표시
- 위치: 플레이백 제어 아래

## 4. 인터랙션 패턴 (Interaction Patterns)

### 4.1 기본 워크플로우
1. 라디오 버튼으로 모드 선택 (정수/문자열, 해싱 방법)
2. 텍스트 입력 필드에 값 입력
3. 버튼 클릭 (Insert/Delete/Find)
4. 애니메이션으로 동작 시각화
5. 애니메이션 제어 버튼으로 단계별 재생/일시정지

### 4.2 사용자 입력 방식
- **직접 입력**: 텍스트 필드에 값 타이핑
- **버튼 선택**: 라디오 버튼, 제어 버튼 클릭
- **슬라이더**: 애니메이션 속도 조절
- **숫자 입력**: 캔버스 크기 설정 (w, h)

### 4.3 피드백 메커니즘
- **시각적 애니메이션**: 데이터 구조 변화를 실시간으로 표시
- **상태 텍스트**: "Animation Completed" 등의 메시지
- **버튼 활성화/비활성화**: 현재 상태에 따른 제어 버튼 활성화

---

## 5. UI 설계 원칙

### 5.1 색상 체계
- **배경**: 녹색 (헤더 및 푸터)
- **제목 텍스트**: 노란색 (Open Hashing, Closed Hashing)
- **입력/제어 영역**: 연한 녹색/흰색
- **시각화 캔버스**: 흰색 배경

### 5.2 레이아웃 구조
- **상단**: 제목 + 입력/제어 영역 (고정)
- **중앙**: 시각화 캔버스 (가변 크기)
- **하단**: 애니메이션 제어 + 설정 (고정)

### 5.3 접근성 특징
- 라디오 버튼으로 명확한 옵션 선택
- 버튼 라벨이 명확함 (Insert, Delete, Find)
- 단계별 애니메이션으로 이해도 향상
- 텍스트 크기와 대비도 적절

---

## 6. 기술적 구현 요소

### 6.1 입력 처리
- HTML text input 필드 (다양한 값 입력 가능)
- JavaScript로 입력값 검증 (정수/문자열)
- 입력 필드 재사용 (Insert, Delete, Find에서 동일 필드 사용)

### 6.2 캔버스 기반 시각화
- HTML5 Canvas 기반 렌더링
- JavaScript로 실시간 그래픽 그리기
- 애니메이션 프레임 기반 동작

### 6.3 상태 관리
- 현재 데이터 구조 상태 유지
- 애니메이션 단계별 상태 추적
- 플레이백 위치 기록

---

## 7. 중요 발견사항

### 7.1 효과적인 학습 설계
- **명확한 구조**: 제어 영역과 시각화 영역 분리
- **단계별 학습**: Skip Back/Step Forward로 각 단계 재검토 가능
- **여러 모드 지원**: 정수/문자열, 다양한 충돌 해결 방법
- **크기 조절 가능**: 사용자 화면에 맞게 캔버스 크기 변경

### 7.2 인터페이스 일관성
- 두 시각화(Open/Closed Hashing) 모두 동일한 제어 구조
- 버튼 배치와 레이아웃이 유사
- 라디오 버튼으로 옵션 그룹화

### 7.3 실용성
- 웹 기반으로 별도 소프트웨어 설치 불필요
- 반응형 캔버스 크기 조절
- 시각화 속도 조절로 개인 학습 속도에 맞춤

---

## 8. 결론

USFCA의 Data Structures Visualization 도구는 **교육 중심 UI/UX 설계**를 보여줌:
- 명확하고 직관적인 제어 구조
- 시각적으로 효과적인 데이터 표현
- 사용자의 학습 속도에 맞는 유연한 애니메이션 제어
- 웹 기반의 접근성과 확장성

이는 인터랙티브 자료구조 튜토리얼 사이트 설계에 참고할 수 있는 우수한 사례이다.
